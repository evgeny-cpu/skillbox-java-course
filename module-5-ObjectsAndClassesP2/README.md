Практическая работа: Инкапсуляция и иммутабельные классы
Описание проекта
Этот проект демонстрирует принципы объектно-ориентированного программирования в Java: инкапсуляцию, создание иммутабельных классов и механизм копирования объектов. Проект состоит из двух заданий, реализующих симуляцию лифта и систему управления грузами для курьерской службы.

Структура проекта
Задание 1: Класс Elevator
Класс Elevator имитирует работу пассажирского лифта с полным контролем доступа к внутреннему состоянию.

Основные компоненты:

Приватные поля: currentFloor, minFloor, maxFloor

Публичные методы для управления лифтом с валидацией вводимых данных

Функциональность:

Конструктор с параметрами минимального и максимального этажей

Методы для перемещения на один этаж вверх/вниз

Метод move(int floor) для перемещения на целевой этаж с пошаговой анимацией

Метод getCurrentFloor() для получения текущего положения лифта

Задание 2: Система управления грузами
Класс Dimensions (Иммутабельный)
Хранит габаритные размеры груза:

Поля: width, height, length (тип double)

Метод calculateVolume() для вычисления объёма

Конструктор с параметрами всех размеров

Класс Cargo (Иммутабельный)
Хранит полную информацию о грузе:

Поля:

dimensions (объект Dimensions)

weight (масса, double)

deliveryAddress (адрес доставки, String)

canBeFlipped (можно переворачивать, boolean)

registrationNumber (регистрационный номер, String)

isFragile (хрупкий ли груз, boolean)

Методы для создания копий с изменёнными полями:

withDeliveryAddress(String newAddress)

withDimensions(Dimensions newDimensions)

withWeight(double newWeight)

Особенности реализации

Пошаговое перемещение с выводом текущего этажа

Иммутабельность в системе грузов

Нет сеттеров для изменения полей

Защита от неконтролируемого изменения состояния
